Java Collections Framework - Interface Hierarchy (Detailed)

Collection<E>
Since: JDK 1.2
The root interface in the collection hierarchy. Represents a group of objects.
Key Methods:
  • add(E e)
  • remove(Object o)
  • contains(Object o)
  • size()
  • isEmpty()
  • iterator()
  • toArray()
  • clear()
  • addAll(Collection<? extends E> c)
  • removeAll(Collection<?> c)
  • retainAll(Collection<?> c)
  • containsAll(Collection<?> c)
Note:

Typically implemented via its sub-interfaces. No direct concrete implementations are provided by the JDK for Collection itself.

List<E>
Since: JDK 1.2
Extends Collection. An ordered collection (sequence). Allows duplicate elements and nulls. Elements can be accessed by their integer index.
Key Methods:
  • get(int index)
  • set(int index, E element)
  • add(int index, E element)
  • remove(int index)
  • indexOf(Object o)
  • lastIndexOf(Object o)
  • listIterator()
  • subList(int fromIndex, int toIndex)
  • All methods from Collection.
Common Implementations:
  • ArrayList: Array-backed list implementation
  • LinkedList: Doubly-linked list implementation
Set<E>
Since: JDK 1.2
Extends Collection. A collection that contains no duplicate elements. Models the mathematical set abstraction.
Key Methods (Behavioral):
  • add(E e)
  • All methods from Collection, with the contract that duplicates are not allowed.
Common Implementations:
  • HashSet: Hash table based set implementation
  • TreeSet: Red-Black tree based set implementation
SortedSet<E>
Since: JDK 1.2
Extends Set. A Set that maintains its elements in ascending order, either naturally or by a specified Comparator.
Key Methods:
  • comparator()
  • first()
  • last()
  • headSet(E toElement)
  • tailSet(E fromElement)
  • subSet(E fromElement, E toElement)
  • All methods from Set.
Common Implementations:
  • ArrayList: Array-backed list implementation
  • LinkedList: Doubly-linked list implementation
  • HashSet: Hash table based set implementation
  • TreeSet: Red-Black tree based set implementation
NavigableSet<E>
Since: JDK 1.6
Extends SortedSet. Provides navigation methods for finding elements closest to a given target, and methods for descending views.
Key Methods:
  • lower(E e)
  • floor(E e)
  • ceiling(E e)
  • higher(E e)
  • pollFirst()
  • pollLast()
  • descendingSet()
  • descendingIterator()
  • All methods from SortedSet.
Common Implementations:
  • ArrayList: Array-backed list implementation
  • LinkedList: Doubly-linked list implementation
  • HashSet: Hash table based set implementation
  • TreeSet: Red-Black tree based set implementation
Queue<E>
Since: JDK 1.5
Extends Collection. A collection designed for holding elements prior to processing. Typically FIFO (First-In, First-Out).
Key Methods:
  • **Adding:** add(E e), offer(E e).
  • **Removing:** remove(), poll().
  • **Examining:** element(), peek().
  • All methods from Collection.
Common Implementations:
  • ArrayList: Array-backed list implementation
  • LinkedList: Doubly-linked list implementation
  • HashSet: Hash table based set implementation
  • TreeSet: Red-Black tree based set implementation
Deque<E>
Since: JDK 1.6
Extends Queue. A double-ended-queue that supports element insertion, removal and inspection at both ends. Can be used as both a queue (FIFO) and a stack (LIFO).
Key Methods (examples for both ends):
  • First End: addFirst(E e), removeFirst(), getFirst()
  • Last End: addLast(E e), removeLast(), getLast()
  • Stack Methods: push(E e), pop()
  • All methods from Queue and Collection
Common Implementations:
  • ArrayList: Array-backed list implementation
  • LinkedList: Doubly-linked list implementation
  • HashSet: Hash table based set implementation
  • TreeSet: Red-Black tree based set implementation
Map<K,V>
Since: JDK 1.2
An object that maps keys to values. A map cannot contain duplicate keys; each key can map to at most one value.
Key Methods:
  • put(K key, V value)
  • get(Object key)
  • remove(Object key)
  • containsKey(Object key)
  • containsValue(Object value)
  • size()
  • isEmpty()
  • clear()
  • keySet()
  • values()
  • entrySet()
  • putAll(Map<? extends K, ? extends V> m)
  • getOrDefault(Object key, V defaultValue) (JDK 8)
  • forEach(BiConsumer<? super K, ? super V> action) (JDK 8)
  • replaceAll(BiFunction<? super K, ? super V, ? extends V> function) (JDK 8)
Common Implementations:
  • **Note:** No direct concrete implementations of Map are provided by the JDK. Instead, it is implemented via its sub-interfaces.
    • Example:
  • HashMap: **Hash table backed map.** Provides fastest access (average $O(1)$) for insertion, deletion, and lookup, but does not guarantee order.
  • LinkedHashMap: **Hash table and linked list backed map.** Maintains insertion order of keys while offering near constant-time performance.
  • (TreeMap implements SortedMap)
  • Hashtable: **Legacy, synchronized hash table.** Similar to HashMap but thread-safe and does not allow null keys or values.
  • EnumMap: Specialized map for enum keys, very efficient.
  • IdentityHashMap: Uses reference equality (==) instead of object equality (equals()) for keys.
  • WeakHashMap: Keys are stored as weak references; entries are garbage collected when their keys are no longer strongly referenced.
SortedMap<K,V>
Since: JDK 1.2
Extends Map. A Map that maintains its key-value mappings in ascending key order, either naturally or by a specified Comparator.
Key Methods:
  • comparator()
  • firstKey()
  • lastKey()
  • headMap(K toKey)
  • tailMap(K fromKey)
  • subMap(K fromKey, K toKey)
  • All methods from Map.
Common Implementations:
  • ArrayList: Array-backed list implementation
  • LinkedList: Doubly-linked list implementation
  • HashSet: Hash table based set implementation
  • TreeSet: Red-Black tree based set implementation
NavigableMap<K,V>
Since: JDK 1.6
Extends SortedMap. Provides navigation methods for finding entries based on closest-match searches for keys, and methods for descending views.
Key Methods:
  • lowerEntry(K key)
  • lowerKey(K key)
  • floorEntry(K key)
  • floorKey(K key)
  • ceilingEntry(K key)
  • ceilingKey(K key)
  • higherEntry(K key)
  • higherKey(K key)
  • firstEntry()
  • lastEntry()
  • pollFirstEntry()
  • pollLastEntry()
  • descendingMap()
  • navigableKeySet()
  • descendingKeySet()
  • All methods from SortedMap.
Common Implementations:
  • ArrayList: Array-backed list implementation
  • LinkedList: Doubly-linked list implementation
  • HashSet: Hash table based set implementation
  • TreeSet: Red-Black tree based set implementation
